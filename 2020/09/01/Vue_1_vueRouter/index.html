hash模式 和 history模式

这两种模式都是客户端实现路由的模式：当页面发生变化的时候，不会向服务端发起请求，只是用JS监听路由的变化，根据不同的地址渲染不同的内容，如果需要服务端的内容会向服务端发送 AJAX 请求。

区别

hash模式：

  1. https://....../#/pla?id=1 ，路径中会带有与数据无关的符号，“#”，“?”…

  2. 原理：基于锚点，以及 onhashchange 事件。

history模式:

  1. https://....../pla/1 ，但需要服务端配置支持。

  2. 原理：基于 HTML5 中的 History API。 history.pushState() （IE10之后支持） / history.replaceState() 将路径放到历史记录里。

vueRouter 实现原理

前端路由，判断前端路径加载不同的组件：

hash 模式：

  1. URL 中 # 后面的内容作为 路径地址。
  2. 监听 hashchange 事件。
  3. 根据当前路由地址找到对应的组件重新渲染。

history 模式：

  1. 路径是普通的URL，通过 history.pushState() 方法改变地址栏，并把当前路径记录到历史记录里，并不会真正的跳转到指定路径，就不会向服务器发送请求。
  2. 监听 popstate 事件，监听浏览器历史记录的变化。
  3. 根据当前路由地址找到对应组件重新渲染。

pushState() / replaceState() 并不会触发该事件。back/forward事件才会被触发。

核心代码分析

VueRouter（对象）：

  1. 属性
    • options 传入的对象 路由规则
    • data 对象，其中的 current 属性记录当前路由，利用 Vue.observable()方法将其设置为响应式的。
  2. 方法
    • Constructor(options):VueRouter 初始化实例属性和方法。
    • _install(Vue):void 静态方法，用来实现vue的插件机制。
    • init():void 为了调用下面三个方法。
    • initEvent():void popstate 事件，监听浏览器历史记录的变化。
    • createRouteMap():void 初始化 routeMap 属性。将传入的路由规则转换为键值对的存储到 routeMap 对象中。路由地址对应组件
    • initComponents(Vue):void 用来创建 router-link 和 router-view 组价的。

问题

报错：

  1. 运行时版：不支持 template 模板，需要打包的时候提前编译。vue-cli 默认为运行时版本，效率更高。
  2. 完整版：包含运行时和编译器，体积比运行时版本大 10k 左右，程序运行时把模板转换为 render 函数。

解决：

  1. 通过配置将运行时版本转为完整版

  2. 通过渲染函数（render()），在运行时版本解决

浏览器前进后退问题

问题：浏览器，前进后退按钮改变地址栏地址，但组件没有响应。

解决：地址栏发生改变，加载响应的组件。利用 popState()会在历史发生变化时触发，pushState() / replaceState() 并不会触发该事件。back/forward事件才会被触发。

详见：initEvent()函数。
