操作系统简介

用户 —>应用程序—>操作系统—>计算机硬件

操作系统：系统调用 、终端命令、图形窗口，即可以在操作系统上安装高级语言的环境，用高级语言开发程序。

操作系统常见分类

  1. 桌面操作系统：Windows 系列、macOS、Linux。
  2. 服务器操作系统：Linux、Windows Server。
  3. 嵌入式操作系统：Linux。
  4. 移动设备操作系统：iOS、Android (基于Linux)。

虚拟机

定义：虚拟机指通过软件模拟的具有完整硬件系统功能的、运行在一个完全隔离环境中的完整的计算机系统。

SSH基础（重点）

SSH（Secure Shell）安全外壳协议，Secure Shell（安全外壳协议，简称SSH）是一种加密的网络传输协议，可在不安全的网络中为网络服务提供安全的传输环境，及创建安全隧道。SSH最常见的用途是远程登录系统，人们通常利用SSH来传输命令行界面和远程执行命令。

Linux之SSH

  1. Linux 中 SSH 是非常常用的工具，通过 SSH客户端 我们可以连接到 SSH服务器 的远程机器上。
  2. 正确的使用SSH可以有效地避免信息泄露（进行加密）、提高传输速度（对数据进行压缩）。

域名端口号

域名是为了方便记忆，端口号可以找到计算机运行的程序。SSH服务器的默认端口号是 22。

常见服务端口号：

  1. SSH服务器：22
  2. Web服务器：80
  3. HTTPS：445
  4. FTP 服务器：21

SSH 客户端的简单使用

Linux 大多自带SSH，可在终端命令使用。Windows 系统需要安装客户端软件PuTTY、XShell、MobaXterm。

exit 退出当前用户的登录。

SCP（掌握）

安全复制（英语：Secure copy，缩写SCP）是指在本地主机与远程主机或者两台远程主机之间基于Secure Shell（SSH）协议安全地传输电脑文件。

-P P要大写。

windows 中可以安装 PuTTY，使用pscp命令传输，或者安装FileZilla使用FTP进行文件传输。

FileZilla 在传输文件时，使用的是 FTP服务而不是SSH服务，因此端口号应该设置为21。

SSH高级（知道）

有关 SSH 配置信息都保存在用户目录下的.ssh目录下

免密码登录

  1. 配置公钥

执行 ssh-keygen 即可生成 SSH 钥匙，一路回车即可

  1. 上传公钥到服务器

执行 ssh-copy-id -p port user@remote 可以让远程服务器记住我们的公钥。

原理

  1. 第一条命令在本地SSH客户端生成两条秘钥（公钥、私钥）。
  2. 第二条命令将公钥复制到SSH服务器。
  3. 本地使用私钥对数据进行加密、解密。
  4. 服务器使用公钥对数据进行加密、解密。

非对称加密算法：

  1. 使用公钥加密数据，需要使用私钥解密。
  2. 使用私钥加密数据，则需要使用公钥解密。

用户权限命令拓展

基本概念

权限

  1. 用户管理包括 用户 与 组 管理。
  2. 每个系统都拥有一个账号，对于不同的系统资源拥有不同的使用权限。
  3. 每一个用户针对不同的文件或目录也有不同的权限。
  4. 这些不同的权限包括：
权限 英文 缩写 数字代号
读 read r 4
写 write w 2
执行 excute x 1

./my.py 表示执行当前文件夹下 my.py 文件。

组

给用户分组，方便进行用户管理。

文件/目录

-rwxrw-r-- 1 user user 43 5月 9 00:00 user.txt

  1. - 开头代表是文件。
  2. d 开头代表是目录。
  3. 接下来三个分为一组，一共三组，分别为：拥有者权限、用户组权限、其他用户权限。
  4. 接下来是硬链接数（cd 可以切换到该文件的方式种数），子目录越多硬链接数越大。
  5. 拥有者、组、大小、时间、名称。

权限修改chmod（重要）

  1. chmod 可以修改 用户、组 对文件、目录的权限。
  2. 命令格式为：chmod +/-rwx 文件名|目录。

超级用户

  1. Linux系统中的root账号通常用于系统的维护和管理，对操作系统的所有资源都具有访问权限。
  2. 不推荐直接使用 root 账号登录系统。
  3. 在Linux安装过程中，系统会自动创建一个用户账号，这个默认用户称为“标准用户”。

sudo

  1. su是 substitute user的缩写，表示使用另一个用户身份。
  2. sudo 用来以其他身份来执行命令，预设的身份为 root。
  3. 用户使用 sudo 必须先输入密码，有5分钟的有效期，过期限必须重新输入密码。

未授权的用户使用sudo ，会发出警告邮件给管理员。

bin和sbin

在 Linux 中，绝大多数可执行文件都保存在/bin、sbin、/usr/bin、/usr/sbin。

  1. /bin（binary）是二进制执行文件目录，主要用于具体应用。
  2. /sbin（system binary）是系统管理员专用的二进制代码存放目录、主要用于系统管理。
  3. /usr/bin （user commands for applications）后期安装的一些软件。
  4. /usr/sbin（super user commands for applications）超级用户的一些管理程序。

  2. ln创建链接时，如果没有 -s选项，则会创建一个硬链接文件。
  3. ls -l可以查看硬连接数。
  4. 相当于复制了一份文件名指向文件数据，源文件删除了，依然可以访问硬链接文件。

在Linux中，文件名和文件的数据是分开存储的，文件名指向文件数据。

创建软链接文件名—>软链接文件数据—>文件名—>文件数据。

打包压缩

在不同的操作系统中，常用的打包压缩方式是不同的：

Windows 常用 rar、Mac常用zip、Linux常用tar.gz。

打包/解包

tar 是 Linux 中最常用的备份工具，这个命令可以把一系列文件打包到一个大文件中，也可以把打包文件恢复成一系列文件。

  1. c ，生成档案文件，创建打包文件。
  2. x ，解开档案文件。
  3. v ，列出归档解档的详细过程，显示进度。
  4. f ，指定档案文件名称，f后面一定是.tar文件，所以必须放在选项最后。

f 要放在最后

压缩/解压缩

gzip

  1. tar与gzip命令结合使用，实现文件 打包和压缩。
  2. tar命令中有个选项-z可以调用gzip，从而可以方便的实现压缩和解压缩的功能。

在Linux中，最常见的压缩文件格式就是xxx.tar.gz。

-C 解压到指定路径，该目录必须存在。

bzip2（two）

  1. 和 gzip 用法一样，扩展名为xxx.tar.bz2。
  2. 在tar命令中用-j选项调用bzip2来实现。
