最简单的部署方式

  1. 配置 Host + Port。
  2. 压缩发布包。
  3. 把发布包传到服务端（利用FTP、Git、Linux SCP命令等）。
  4. 服务端解压。
  5. 安装依赖。
  6. 启动服务。

配置 nuxt.config.js：

向服务端传递压缩资源：

  1. 传递的资源文件：
    • .nuxt文件（nuxt打包生成的资源文件）。
    • static文件（静态资源）。
    • nuxt.config.js文件（给nuxt服务使用的配置文件）。
    • package.json文件（安装依赖使用）。
    • package-lock.json文件。
  2. 压缩文件。
  3. 连接服务器。
  4. 利用SCP命令上传文件。
  5. 连接服务器解压。
  6. 安装依赖。
  7. npm run start 启动Web服务。
  8. 访问（使用公网IP地址而非服务器局域网）。

使用PM2启动Node服务

使用PM2在后台运行应用解决 node 启动占用命令行问题：
PM2是node进程管理工具，可以利用它来简化很多node应用管理的繁琐任务，如性能监控、自动重启、负载均衡等，使用简单。
[https://pm2.io]

首屏渲染：
首次屏幕的渲染，其它的交由前端交互渲染。

jsonplaceholder：
提供了假接口的网站数据接口。

需要通过接口展示并且是通过服务端渲染：
写到Nuxt提供的特殊的生命钩子 asyncDate 中，首屏访问到这个数据会在服务端请求渲染为静态的html返回，非首屏在客户端请求渲染。
想要只在客户端渲染直接放到 mounted 中。

改造简单vueCli项目，仅首屏渲染：
不提倡。
建议：手动搭建Vue SSR开发环境，成本基本相同。
如果首页中没有需要动态变化的数据，可以考虑使用预渲染的方案，[https://github.com/chrisvfritz/prerender-spa-plugin]。

预渲染：
就是提前把组件把组件编译为纯静态的HTML文件。有动态数据的还是采用服务端渲染。
